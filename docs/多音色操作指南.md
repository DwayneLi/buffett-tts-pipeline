# 多音色操作指南：批量翻译 + 巴菲特/芒格双音色合成

> 适用：你已有"带听众提问、姓名、巴菲特/芒格角色标注"的整理稿，要完成 ① 大模型批量翻译 ② 双音色中文语音合成。
> 本次改动把项目从 PRD v1.0 单音色扩展到 v1.1 多音色（巴菲特 + 芒格）。
> 对应 PRD：F2 翻译、F4 分段、F5/F6 多音色合成（v1.1）。

## 前置检查

| 依赖 | 第一步翻译 | 第二步合成 |
|------|:---:|:---:|
| GLM / 千问 API Key | ✅ 必须 | ✅（翻译用）|
| Windows GPT-SoVITS 服务（`api_v2.py` :9880）| ❌ 不需要 | ✅ 必须 |
| 巴菲特参考音频（5–15s，16kHz 单声道）| ❌ | ✅ 必须 |

## 第一步：大模型批量翻译

只依赖 LLM API（GLM/千问），**不需要 Windows GPU**，现在就能做。

### 1.1 准备输入
确保整理稿里发言者角色清晰，例如：
```
Question (John Smith, ABC News): ...
Warren Buffett: ...
Charlie Munger: ...
```
脚本会按 `Question`/`Q` 或空行切块（每块 ≤4000 字），翻译时由 prompt 重新标注成 `【提问人】【问题】【巴菲特】【芒格】【📌 背景】`。

### 1.2 运行翻译
```powershell
cd C:\Users\donal\OneDrive\projects\audio_factory\buffett-tts-pipeline
python scripts\translate.py 你的整理稿.txt --provider glm --output brk_translated.json
```
- `--provider glm` 可换 `qwen`；或在 `config.json` 的 `llm.provider` 填 `glm`/`qwen`（见 provider 切换说明）。
- 产出：`brk_translated.json` + `brk_translated.txt`（`.txt` 带 `【角色】` 标记，供第二步用）。
- 失败的段会自动重试 3 次（PRD F2-04）。

### 1.3 验证
打开 `brk_translated.txt`，确认：
- 巴菲特、芒格的发言分别带 `【巴菲特】`、`【芒格】` 标记，没有混并；
- 每个问答之间用 `---` 分隔；
- 金融术语准确、口语化、背景补充适量。

> 建议先用一小段试翻译，确认角色标注正确再跑全量。

## 第二步：双音色中文语音合成

依赖：Windows 上 GPT-SoVITS `api_v2.py` 已启动（端口 9880），且巴菲特 + 芒格参考音频已就绪。

### 2.1 收集芒格参考音频
和巴菲特同样的流程（见 `docs/Windows素材收集指南.md`）：
- 下一段芒格单人讲话视频 → `ffmpeg` 转 16kHz 单声道 WAV → 裁 5–15 秒干净片段。
- 命名如 `munger_ref_10s.wav`，放到 `C:\Users\donal\OneDrive\projects\audio_factory\`。
- 可选：再准备一段提问人/旁白参考音频 `narrator_ref_10s.wav`（没有的话提问段会被跳过，只合成巴菲特+芒格）。

### 2.2 配置 config.json 的多音色映射
在 `config/config.json` 的 `tts` 段加 `ref_audio_paths`：
```json
"tts": {
  "windows_host": "你的Windows局域网IP",
  "api_port": 9880,
  "ref_audio_paths": {
    "buffett": "C:\\Users\\donal\\OneDrive\\projects\\audio_factory\\buffett_ref_10s.wav",
    "munger":  "C:\\Users\\donal\\OneDrive\\projects\\audio_factory\\munger_ref_10s.wav",
    "narrator":"C:\\Users\\donal\\OneDrive\\projects\\audio_factory\\narrator_ref_10s.wav"
  }
}
```
> `background`（背景补充）默认复用 `narrator` 的音色；不配 `narrator` 则提问段和背景段都会被跳过。

### 2.3 运行端到端流水线
```powershell
python scripts\pipeline.py 你的整理稿.txt --provider glm
```
脚本会自动：
1. **翻译**（调 GLM/千问，产出带角色标记的译文）；
2. **按角色拆分**成逐句 utterance（提问人+问题合并为 narrator，巴菲特/芒格各自成段，超 500 字自动切分）；
3. **按角色选参考音频**逐段调 Windows GPT-SoVITS `/tts` 合成（巴菲特段用 buffett ref，芒格段用 munger ref）；
4. **ffmpeg 拼接** + 段间 1.5s 静音 → 输出 MP3。

启动时会打印"音色配置"清单，确认每个角色都配了 ref。

### 2.4 也可纯命令行指定参考音频（不改 config）
```powershell
python scripts\pipeline.py 你的整理稿.txt --provider glm `
  --ref-audio-buffett "C:\...\buffett_ref_10s.wav" `
  --ref-audio-munger  "C:\...\munger_ref_10s.wav" `
  --ref-audio-narrator "C:\...\narrator_ref_10s.wav"
```

### 2.5 验证
- 听输出 MP3：巴菲特段是巴菲特音色、芒格段是芒格音色、提问段是旁白音色；
- 角色分布可在 Step 2 的日志里看到（如 `{'narrator': N, 'buffett': M, 'munger': K, 'background': L}`）；
- 断点续传：中断后重跑，已合成的 `seg_xxxx.wav` 会自动跳过。

## 常见问题

| 现象 | 解决 |
|------|------|
| 译文里没有 `【芒格】` 标记 | 整理稿里芒格发言角色不清晰；在输入里明确标 `Charlie Munger:` 再重翻 |
| 芒格段被跳过 | 没配 `ref_audio_paths.munger`（或 `--ref-audio-munger`）|
| 提问段没声音 | 没配 narrator 参考音频；不想听提问可忽略，或配一个旁白 ref |
| 合成报连不上 | 确认 Windows 上 `python api_v2.py` 已起、防火墙放行 9880、IP 正确 |
| 单段超时 | 已自动按 500 字切分；仍超时可调小 `MAX_UTTERANCE_CHARS` |

2. **再补芒格音频** + 启动 Windows GPT-SoVITS。
